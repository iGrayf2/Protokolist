# Protokolist

> Превращает длинную запись производственного совещания в готовый деловой протокол Word.

Protokolist — это Ubuntu-first CLI-конвейер для регулярной обработки совещаний: он распознает аудио через Faster-Whisper, сохраняет RAW-данные, делает мягкую очистку текста, собирает специальный пакет для ChatGPT и приводит ChatGPT к одному стабильному формату итогового протокола.

Главная идея проекта: не просто получить стенограмму, а быстро получить документ, который можно читать, переслать руководителю и использовать как протокол совещания.

## Что получается на выходе

Итоговый результат после загрузки пакета в ChatGPT:

```text
Протокол совещания
Документ составлен по доступной части стенограммы (~N минут).

Присутствующие
• Андрей Аркадьевич — ведущий совещания
• Катерина — оператор 1С
• Мария — мастер смены
• Алла — мастер смены
• Ольга — мастер смены
• Остальные участники

Тезисная стенограмма
[00:00] Андрей Аркадьевич: Объявляет тестирование системы записи и распознавания совещаний.
[00:01:36] Катерина: Докладывает итоги июня по складу.
[00:12:10] Мария: Представляет данные по браку.

Основные темы
• Тестирование системы протоколирования.
• Итоги склада за июнь.
• Пересчет остатков вязального цеха.
• Учет брака.

```

ChatGPT должен вернуть файл:

```text

```


## Почему не просто аудио в ChatGPT

Длинные совещания неудобно обрабатывать напрямую: много текста, шум, повторы Whisper, неясные говорящие, производственные термины, риск выдуманных задач и участников.

Protokolist решает это так:

- сначала локально готовит качественные данные;
- сохраняет RAW с таймкодами;
- делает очищенный текст;
- добавляет профиль предприятия и словарь терминов;
- формирует строгий `CHATGPT_TASK.md`;
- заставляет ChatGPT выдавать один и тот же короткий формат протокола.

## Быстрый старт на Ubuntu

Рекомендуемое расположение:

```text
~/Protokolist/
```

Установка:

```bash
cd ~
git clone https://github.com/iGrayf2/Protokolist.git
cd Protokolist
chmod +x scripts/*.sh
./scripts/install_ubuntu.sh
```

Проверка окружения:

```bash
PYTHONPATH=src .venv/bin/python -m protokolist.cli doctor
```

Команда проверит Python, ffmpeg, зависимости, рабочие папки, профиль предприятия, словарь, свободное место и наличие `nvidia-smi`.

## Обработка записи

Положи аудио или видео в папку `input/`:

```text
input/meeting.mp3
```

Запусти обработку:

```bash
./scripts/transcribe_file_ubuntu.sh input/meeting.mp3
```

По умолчанию используется быстрый CPU-режим:

```text
model: small
device: cpu
compute_type: int8
```

После завершения открой папку `output/`. Там появится отдельная папка встречи и ZIP-пакет для ChatGPT:

```text
output/<дата>_<имя_записи>/
├── CHATGPT_PACKAGE/
├── <имя>_chatgpt_package.zip
└── artifacts/
```

Главный файл для пользователя:

```text
<имя>_chatgpt_package.zip
```

## Использование с ChatGPT

1. Открой ChatGPT.
2. Загрузи `<имя>_chatgpt_package.zip`.
3. Напиши:

```text
Выполни CHATGPT_TASK.md
```

4. Получи готовый Word-документ `protokol_soveshchaniya.docx`.

## Модели Whisper

### Быстро и достаточно хорошо

```bash
./scripts/transcribe_file_ubuntu.sh input/meeting.mp3
```

Это автоматически использует:

```text
--model small
```

Подходит для регулярных совещаний, когда важна скорость.

### Баланс качества и скорости

```bash
./scripts/transcribe_file_ubuntu.sh input/meeting.mp3 --model medium
```

Хороший вариант, если `small` слишком часто ошибается на именах, цифрах или терминах.

### Максимальное качество, но медленно

```bash
./scripts/transcribe_file_ubuntu.sh input/meeting.mp3 --model large-v3
```

Используй для важных записей, где скорость не критична.

## Примерные режимы

| Модель | Скорость на CPU | Качество | Когда использовать |
|---|---:|---:|---|
| `small` | высокая | нормальное | обычные совещания, быстрый результат |
| `medium` | средняя | хорошее | если важны имена, цифры и термины |
| `large-v3` | низкая | максимальное | финальные или особо важные записи |

## Что находится в CHATGPT_PACKAGE

```text
CHATGPT_PACKAGE/
├── CHATGPT_TASK.md
├── README.md
├── meeting_context.json
├── meeting.raw.json
├── meeting.raw.txt
├── meeting.cleaned.txt
├── meeting.cleaned_with_timestamps.txt
├── meeting.quality_report.txt
└── meeting.quality_report.json
```

### CHATGPT_TASK.md

Главная инструкция для ChatGPT. Она жестко требует короткий Word-протокол в формате:

- присутствующие;
- тезисная стенограмма;
- основные темы;
- принятые решения;
- таблица задач из 3 колонок: `Задача`, `Ответственный`, `Срок`.

Шаблон инструкции хранится отдельно:

```text
templates/CHATGPT_TASK.md
```

Его можно править без изменения Python-кода.

### meeting_context.json

Профиль предприятия и правила обработки:

- возможные участники;
- роли;
- мастера смен;
- производственный контекст;
- термины;
- правила неуверенности.

Важно: профиль — это справочник, а не доказательство присутствия человека на встрече. ChatGPT не должен добавлять людей в протокол только потому, что они есть в профиле.

### meeting.raw.json

Главный источник правды. Содержит:

- сырые сегменты Whisper;
- начало и конец каждого сегмента;
- слова с таймкодами;
- вероятности;
- параметры модели;
- язык;
- длительность;
- метаданные.

### meeting.cleaned.txt

Текст после мягкой словарной очистки. RAW при этом не изменяется.

### meeting.quality_report.txt/json

Отчет качества распознавания. Сейчас проверяет:

- расшифровку, которая закончилась заметно раньше конца аудио;
- повторяющиеся фразы в конце, которые могут быть галлюцинацией Whisper на тишине или шуме.

## Что находится в artifacts

`artifacts/` — служебная папка для разработчика и ручной проверки:

```text
artifacts/
├── prepared_audio.wav
├── meeting.raw.json
├── meeting.raw.txt
├── meeting.cleaned.txt
├── meeting.cleaned.srt
├── meeting.cleaned.docx
├── meeting.quality_report.txt
└── meeting.quality_report.json
```

Эти файлы обычно не нужно отправлять в ChatGPT вручную: все нужное уже собрано в `CHATGPT_PACKAGE` и ZIP.

## Предобработка аудио

Перед Whisper по умолчанию создается подготовленный файл:

```text
artifacts/prepared_audio.wav
```

Параметры:

```text
mono WAV 16 kHz
```

Исходная запись не изменяется.

Отключить предобработку можно так:

```bash
./scripts/transcribe_file_ubuntu.sh input/meeting.mp3 --no-preprocess
```

Подробности: `docs/audio_preprocessing.md`.

## Запуск длинной обработки через tmux

Для длинных записей лучше запускать обработку в `tmux`, чтобы SSH-сессия или терминал не оборвали процесс.

Установить `tmux`:

```bash
sudo apt install -y tmux
```

Создать сессию:

```bash
cd ~/Protokolist
tmux new -s protokolist
```

Запустить обработку:

```bash
./scripts/transcribe_file_ubuntu.sh input/meeting.mp3
```

Отключиться, не останавливая процесс:

```text
Ctrl+B
D
```

Вернуться:

```bash
tmux attach -t protokolist
```

Полная памятка: `docs/tmux.md`.

## Структура проекта

```text
Protokolist/
├── input/                 # сюда кладутся аудио/видео записи
├── output/                # сюда сохраняются результаты обработки
├── logs/                  # будущие логи
├── models/                # будущие локальные модели/кэш
├── cache/                 # временные данные
├── config/                # словари и настройки
├── profiles/              # профили предприятий
├── templates/             # шаблоны инструкций для ChatGPT
├── scripts/               # установочные и запускные скрипты
├── src/protokolist/        # код приложения
└── README.md
```

Папки `input`, `output`, `logs`, `models`, `cache` есть в репозитории только как пустая структура. Реальные записи, результаты, модели и логи не попадают в Git.

## Словарь исправлений

Словарь находится здесь:

```text
config/dictionary.json
```

Он помогает сохранять производственные термины и исправлять типовые ошибки распознавания.

## Профиль предприятия

Основной профиль:

```text
profiles/factory_moydod.json
```

Он хранит контекст предприятия, возможных участников, роли и производственные термины.

## Текущий фокус

Сейчас главный фокус проекта — стабильный повторяемый результат:

```text
запись совещания → ZIP для ChatGPT → короткий протокол Word
```

Не цель: идеальная дословная стенограмма.

Цель: понятный протокол, решения и задачи без выдуманных фактов.
