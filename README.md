# Protokolist

> Локальная система подготовки **максимально качественной стенограммы и протокола совещания** с использованием Faster-Whisper и LLM.

---

# Идея проекта

Большинство программ делают одно:

**Аудио → Текст**

Protokolist делает иначе.

Он старается сохранить **максимум информации**, чтобы затем искусственный интеллект смог восстановить смысл разговора максимально точно.

Конвейер выглядит так:

```text
Аудио
   │
   ▼
Faster-Whisper (large-v3)
   │
   ▼
RAW
(raw.json + raw.txt)
   │
   ▼
Очистка словарем предприятия
   │
   ▼
cleaned.txt
cleaned.docx
cleaned.srt
   │
   ▼
Генерация специализированных промптов
   │
   ├── 01 Анализ встречи
   ├── 02 Исправление распознавания
   ├── 03 Определение говорящих
   ├── 04 Финальный протокол
   └── 05 Контроль качества
   │
   ▼
LLM
(ChatGPT / OpenAI / локальная модель)
   │
   ▼
Профессиональная стенограмма
Протокол
Поручения
Решения
Action Items
```

---

# Почему не просто Whisper?

Whisper отлично распознает речь.

Но он:

* не знает сотрудников предприятия;
* не понимает структуру организации;
* не знает терминологию;
* не умеет надежно определять говорящих;
* не может составить качественный протокол совещания.

Поэтому задача Protokolist — подготовить максимально качественные исходные данные.

---

# Основные возможности

* Faster-Whisper
* Максимальное качество (`large-v3`)
* Word timestamps
* Сохранение сырого результата
* Очистка словарем предприятия
* Экспорт TXT
* Экспорт DOCX
* Экспорт SRT
* Подготовка промптов для LLM
* Профили предприятий
* Работа полностью локально

# Быстрый старт

Клонировать репозиторий:

```bash
git clone https://github.com/iGrayf2/Protokolist.git
cd Protokolist
```

Установить:

```bat
scripts\install_windows.cmd
```

Запустить GUI:

```bat
scripts\run_gui.cmd
```

Выбрать аудио.

Нажать **Transcribe**.

# RAW — главный источник правды

Самый важный файл проекта —

```
meeting.raw.json
```

Он содержит:

* исходный текст Whisper;
* начало и конец каждого сегмента;
* слова с таймкодами;
* вероятности;
* параметры модели;
* язык;
* метаданные.

RAW никогда не изменяется.

Все исправления выполняются поверх него.

---

# Cleaned

После сохранения RAW применяется словарь предприятия.

Исправляются:

* названия систем;
* имена сотрудников;
* технические термины;
* типичные ошибки распознавания.

RAW при этом остается неизменным.

---

# Конвейер LLM

Protokolist специально не пытается выполнять всю работу одним запросом.

Используется последовательность специализированных этапов.

## 01 — Анализ встречи

Определяются:

* ведущий;
* участники;
* темы;
* термины;
* проблемные места.

---

## 02 — Исправление распознавания

Исправляются только очевидные ошибки распознавания.

Смысл разговора не меняется.

---

## 03 — Определение говорящих

На основании контекста определяется:

* кто говорит;
* роли участников;
* неопознанные говорящие.

---

## 04 — Финальный протокол

Формируются:

* присутствующие;
* повестка;
* обсуждение;
* решения;
* поручения;
* сроки;
* ответственные.

---

## 05 — Контроль качества

Дополнительная проверка ищет:

* потерянные реплики;
* ошибки в именах;
* ошибки в числах;
* противоречия;
* сомнительные места.

---

# Профили предприятий

Каждое предприятие может иметь собственный профиль.

Например:

```
profiles/
    factory_moydod.json
```

Профиль хранит:

* сотрудников;
* должности;
* словарь терминов;
* производственный контекст;
* правила определения участников;
* особенности конкретного предприятия.

Благодаря этому Protokolist легко адаптируется под новые организации без изменения кода.

---

# Словарь

```
config/dictionary.json
```

Используется двумя способами:

1. как подсказка для Whisper;
2. как постобработка распознанного текста.
