# CHATGPT_TASK.md

# Задача

Ты получил пакет Protokolist с материалами производственного совещания.

Твоя задача — внимательно проанализировать ВСЕ материалы и создать короткий деловой протокол совещания в формате Microsoft Word (.docx).

Главная цель — не скорость, а полнота и достоверность.

Запрещено делать выводы по первым минутам записи или по одному файлу.

------------------------------------------------------------
ОЖИДАЕМЫЙ РЕЗУЛЬТАТ
------------------------------------------------------------

Создай файл

protokol_soveshchaniya.docx

После создания файла ответь в чате только двумя строками:

Готово.

[ссылка на protokol_soveshchaniya.docx]

Если создать Word невозможно —
честно сообщи об этом и выведи тот же протокол в Markdown.

Не выводи дополнительные пояснения.

------------------------------------------------------------
ВХОДНЫЕ ФАЙЛЫ
------------------------------------------------------------

В пакет могут входить:

meeting.raw.json
meeting.raw.txt
meeting.cleaned.txt
meeting.cleaned_with_timestamps.txt
meeting_context.json

Дополнительно могут присутствовать:

quality_report.json
quality_report.md
README.md
и другие служебные файлы.

------------------------------------------------------------
ПРИОРИТЕТ ИСТОЧНИКОВ
------------------------------------------------------------

Если данные различаются, использовать источники в следующем порядке:

1. meeting.raw.json
2. meeting.cleaned_with_timestamps.txt
3. meeting.cleaned.txt
4. meeting.raw.txt
5. meeting_context.json

meeting_context.json является только справочником.

Он НЕ доказывает присутствие людей и НЕ является источником содержания совещания.

------------------------------------------------------------
ЭТАП 1.
ПРОВЕРКА ПАКЕТА
(ОБЯЗАТЕЛЬНО)
------------------------------------------------------------

Перед началом анализа необходимо изучить все доступные материалы.

Минимум должны быть просмотрены:

✓ meeting.raw.json
✓ meeting.cleaned.txt
✓ meeting.cleaned_with_timestamps.txt
✓ meeting_context.json

Использовать только один файл запрещено.

------------------------------------------------------------
ЭТАП 2.
ПРОВЕРКА ПОЛНОТЫ ЗАПИСИ
(ОБЯЗАТЕЛЬНО)
------------------------------------------------------------

До начала написания протокола необходимо определить:

• время начала записи;

• время окончания записи;

• общую длительность;

• последний существующий таймкод.

После этого необходимо убедиться, что анализ охватывает запись полностью —
от первого до последнего таймкода.

Если стенограмма заканчивается раньше окончания записи,
или пакет явно неполный,
НЕ создавай итоговый протокол.

Вместо этого сообщи пользователю:

— что пакет неполный;

— какой последний найденный таймкод;

— почему невозможно сделать полноценный протокол.

------------------------------------------------------------
ЭТАП 3.
АНАЛИЗ СОВЕЩАНИЯ
------------------------------------------------------------

Перед созданием Word необходимо полностью проанализировать материалы.

Последовательно определить:

1.
Кто участвовал.

2.
Какие темы обсуждались.

3.
Какие решения были приняты.

4.
Какие поручения были выданы.

5.
Какие сроки были названы.

Только после завершения анализа приступать к созданию документа.

------------------------------------------------------------
ОПРЕДЕЛЕНИЕ УЧАСТНИКОВ
------------------------------------------------------------

Имя можно использовать только если имеется прямое основание.

Допустимые основания:

• самопредставление;

• прямое обращение;

• явно продолжающаяся реплика того же человека.

Если говорящий неизвестен —

использовать

Неопознанный участник N

Запрещено писать:

вероятно

предположительно

скорее всего

возможно

Не использовать профиль предприятия для угадывания участников.

------------------------------------------------------------
ТЕМЫ СОВЕЩАНИЯ
------------------------------------------------------------

Совещание должно быть проанализировано полностью.

Если запись содержит несколько тем,
каждая тема должна попасть в итоговый документ.

Допускается сокращать обсуждение.

Запрещено пропускать целые темы.

Последний тезис обязан относиться к последней части записи.

Нельзя заканчивать документ после первых минут разговора.

------------------------------------------------------------
РЕШЕНИЯ
------------------------------------------------------------

Решением считается только:

• принятое решение;

• поручение;

• требование руководителя;

• договоренность;

• утвержденное направление действий.

Если участники только обсуждали вопрос —

не считать это решением.

Раздел

Принятые решения

может быть пустым.

------------------------------------------------------------
ЗАДАЧИ
------------------------------------------------------------

В таблицу задач включать только реальные поручения.

Если ответственный неизвестен —

писать

не определен

Если срок неизвестен —

писать

не определен

Не придумывать задачи.

Если задач нет —

оставить таблицу пустой.

------------------------------------------------------------
ФОРМАТ WORD
------------------------------------------------------------

Документ должен содержать ТОЛЬКО следующие разделы.

1.

Протокол совещания

2.

Документ составлен по полной стенограмме (~XX минут).

или

Документ составлен по доступной части стенограммы (~17 из 42 минут).

Длительность должна вычисляться автоматически.

3.

Присутствующие

4.

Тезисная стенограмма

5.

Основные темы

6.

Принятые решения

7.

Таблица

Задача | Ответственный | Срок

Других разделов быть не должно.

------------------------------------------------------------
ТРЕБОВАНИЯ К ТЕЗИСНОЙ СТЕНОГРАММЕ
------------------------------------------------------------

Использовать короткие деловые формулировки.

Не переписывать разговор дословно.

Каждая запись —

одно краткое смысловое действие.

Например

Докладывает

Просит

Предлагает

Подчеркивает

Подтверждает

Переходит к следующему вопросу

Таймкоды сохранять.

Допустимый формат

[00:15]

или

[00:15:32]

------------------------------------------------------------
ЧЕГО НЕЛЬЗЯ ДЕЛАТЬ
------------------------------------------------------------

Запрещено:

• выдумывать факты;

• выдумывать участников;

• выдумывать решения;

• выдумывать поручения;

• выдумывать сроки;

• назначать ответственных без основания;

• добавлять аналитический отчет;

• добавлять контроль качества;

• добавлять карту встречи;

• добавлять вероятных участников;

• использовать профиль предприятия как доказательство.

------------------------------------------------------------
ФИНАЛЬНАЯ САМОПРОВЕРКА
(ОБЯЗАТЕЛЬНО)
------------------------------------------------------------

Перед созданием Word-файла необходимо убедиться:

✓ просмотрены все входные файлы;

✓ определена полная длительность записи;

✓ анализ выполнен от первого до последнего таймкода;

✓ отражены все основные темы;

✓ отражены последние обсуждения совещания;

✓ не потеряны последние решения;

✓ участники не выдуманы;

✓ решения не выдуманы;

✓ задачи не выдуманы;

✓ сроки не выдуманы;

✓ документ соответствует шаблону.

Только после успешной проверки разрешается создавать итоговый Word-документ.
